# 韩国星际争霸团战数据自动采集与统计平台

## PRD_V1.3（最终版）

------

# 1. 项目概述

## 1.1 项目名称

韩国星际争霸团战数据自动采集与统计平台

------

## 1.2 项目目标

构建一套长期稳定运行的电竞数据平台，实现：

```text
自动采集
    ↓
自动翻译
    ↓
自动统计
    ↓
数据分析
    ↓
网页展示
```

系统主要服务于韩国星际争霸职业团战数据统计与分析。

------

## 1.3 项目定位

本项目定位为：

### 电竞数据平台

核心功能：

- 自动采集比赛数据
- 自动翻译韩文数据
- 选手数据统计
- 奖金数据统计
- 地图数据统计
- 队伍数据统计
- 历史数据查询

------

## 1.4 不包含功能

以下功能不属于V1版本范围：

- AI战报生成
- 微信公众号推送
- 自动发文
- 比赛预测系统
- Elo评分系统

以上功能作为未来扩展预留。

------

# 2. 数据来源

## 2.1 数据源网站

ELOBOARD职业团战页面

主入口：

https://eloboard.com/men/bbs/board.php?bo_table=pro_league

------

## 2.2 数据获取方式

系统每日自动访问列表页。

获取最新文章：

```text
wr_id
标题
发布时间
文章链接
```

示例：

```text
https://eloboard.com/men/bbs/board.php?bo_table=pro_league&wr_id=2455
```

------

## 2.3 唯一标识

系统以：

```text
wr_id
```

作为比赛唯一标识。

数据库中建立唯一索引。

禁止使用标题判断重复数据。

------

# 3. 自动采集需求

## 3.1 自动采集时间

每日执行：

```text
10:00
```

主采集任务。

------

## 3.2 延迟补采机制

若10:00未发现新文章：

自动补查：

```text
10:30
11:00
12:00
```

当天12:00后结束采集。

------

## 3.3 增量采集

流程：

```text
获取最新文章
↓
读取wr_id
↓
检查数据库
↓
若已存在
    结束
↓
若不存在
    开始采集
```

------

# 4. 历史数据回补

## 4.1 回补范围

系统首次部署时：

回补范围：

```text
2026-01-01
至当前日期
```

------

## 4.2 回补模式

后台支持：

```text
指定日期范围
指定wr_id范围
```

重新导入。

------

# 5. 数据采集内容

## 5.1 比赛信息

采集：

```text
比赛日期
比赛标题
wr_id
原始链接
```

------

## 5.2 队伍信息

采集：

```text
队伍名称
队伍积分
队伍奖金
```

（若页面存在）

------

## 5.3 选手信息

采集：

```text
韩文名
游戏ID
种族
所属队伍
```

------

## 5.4 对战记录

采集：

```text
阶段
选手A
选手B
地图
胜者
败者
比分
```

------

## 5.5 奖金数据

采集：

```text
选手
奖金金额
```

系统不推算奖金。

仅保存页面公布结果。

------

# 6. 翻译系统

## 6.1 翻译规则来源

用户提供：

```text
translate_rules.md
```

------

## 6.2 翻译优先级

```text
translate_rules.md
↓
历史数据库
↓
AI翻译
↓
保留韩文
```

------

## 6.3 翻译对象

包含：

```text
选手名称
地图名称
队伍名称
赛事名称
```

------

# 7. 数据冻结原则

## 7.1 首次采集原则

当比赛首次采集成功后：

```text
matches
match_details
prize_pool
```

全部冻结。

------

## 7.2 后续更新策略

即使ELOBOARD修改：

```text
奖金
比分
标题
内容
```

系统不自动覆盖。

保留首次采集结果。

------

# 8. 原始网页存档
