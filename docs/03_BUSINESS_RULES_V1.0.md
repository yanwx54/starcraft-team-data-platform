# BUSINESS_RULES_V1.0

## 项目

韩国星际争霸团战数据自动采集与统计平台

版本：V1.0

状态：需求冻结版

------

# 1. 文档目的

本文件定义项目所有核心业务规则。

优先级高于：

- API_SPEC
- CRAWLER_SPEC
- DATABASE

如出现冲突，以本文件为准。

------

# 2. 数据来源规则

## 2.1 唯一数据源

唯一可信数据源：

ELOBOARD职业团战页面

```text
https://eloboard.com/men/bbs/board.php?bo_table=pro_league
```

------

## 2.2 唯一文章标识

系统使用：

```text
wr_id
```

作为唯一比赛标识。

不得使用：

```text
标题
发布时间
文章URL
```

作为唯一键。

------

## 2.3 数据可信度原则

ELOBOARD首次发布的数据视为唯一可信数据。

系统不参考：

- Liquipedia
- TeamLiquid
- Reddit
- 其他第三方网站

进行数据修正。

------

# 3. 数据冻结规则

## 3.1 首次采集原则

当文章首次采集成功后：

```text
matches
match_details
prize_pool
```

立即冻结。

------

## 3.2 后续修改处理

即使ELOBOARD后续修改：

```text
奖金
比分
队伍信息
标题
正文内容
```

系统均不自动更新。

------

## 3.3 原因

保持统计口径一致。

避免排行榜和历史数据发生变化。

------

## 3.4 原始HTML保存

系统必须永久保存：

```text
raw_articles.html_content
```

用于审计和追溯。

但不得覆盖业务数据。

------

# 4. 团战赛制规则

## 4.1 标准赛制

大部分比赛采用：

```text
第一轮 BO7
第二轮 KOF
```

------

## 4.2 另一种赛制

部分比赛采用：

```text
第一轮 BO7
第二轮 BO7
```

------

## 4.3 ACE决胜局

当两轮比赛结果：

```text
1 : 1
```

时进入：

```text
ACE
```

决胜局。

------

## 4.4 支持赛制

系统必须支持：

```text
BO7
KOF
ACE
```

------

## 4.5 阶段定义

| 类型 | 说明           |
| ---- | -------------- |
| BO7  | 固定七局团队赛 |
| KOF  | 胜者留场擂台赛 |
| ACE  | 决胜局         |

------

# 5. 胜负统计规则

## 5.1 基本原则

统计单位：

```text
单场对局
```

而非团战。

------

## 5.2 个人胜场

每赢下一局：

```text
胜场 +1
```

------

示例：

```text
A连续击败：

B
C
D
```

统计：

```text
胜场 +3
```

------

## 5.3 个人负场

每输掉一局：

```text
负场 +1
```

------

## 5.4 胜率

公式：

```text
胜率 = 胜场 ÷ (胜场 + 负场)
```

------

# 6. 连胜规则

## 6.1 连胜定义

连续赢得个人对局。

------

## 6.2 KOF规则

KOF中：

```text
连续击败多个选手
```

按实际场数累计。

------

示例：

```text
A击败B
A击败C
A击败D
```

结果：

```text
当前连胜 = 3
```

------

## 6.3 连胜终止

任意一场失利：

```text
当前连胜归零
```

------

# 7. 团战统计规则

## 7.1 团战胜负

仅统计：

```text
队伍
```

层面。

------

## 7.2 个人统计独立

个人统计：

```text
胜场
负场
胜率
连胜
```

与团战结果无关。

------

# 8. 奖金规则

## 8.1 奖金来源

奖金以页面公布结果为准。

系统不进行计算。

------

## 8.2 奖金归属

奖金归属于：

```text
选手个人
```

不是队伍。

------

## 8.3 奖金记录

页面公布：

```text
选手
奖金金额
```

系统直接入库。

------

## 8.4 奖金统计

支持：

```text
累计奖金

赛季奖金

月度奖金

奖金排行榜
```

------

# 9. 选手身份规则

## 9.1 唯一身份

唯一身份：

```text
player_uid
```

------

## 9.2 禁止使用

不得使用：

```text
韩文名

中文名

游戏ID
```

作为唯一身份。

------

## 9.3 允许变化

允许：

```text
改ID

改译名

转队
```

历史数据必须保持连续。

------

# 10. 队伍规则

## 10.1 唯一身份

唯一身份：

```text
team_uid
```

------

## 10.2 队伍更名

允许队伍名称变化。

历史数据保留。

------

# 11. 翻译规则

## 11.1 翻译优先级

严格按照：

```text
translate_rules.md
↓
历史数据库
↓
AI翻译
↓
保留韩文
```

执行。

------

## 11.2 选手翻译

同一个：

```text
player_uid
```

必须保持唯一中文名称。

------

## 11.3 地图翻译

同一地图允许：

```text
韩文名
英文名
中文名
```

同时存在。

------

# 12. 地图标准化规则

## 12.1 唯一身份

地图唯一标识：

```text
map_uid
```

------

## 12.2 别名归并

例如：

```text
네오 실피드

Neo Sylphid

NEO SYLPHID
```

必须归并为同一地图。

------

# 13. 赛季规则

## 13.1 赛季管理

所有比赛必须归属：

```text
season_id
```

------

## 13.2 统计隔离

不同赛季统计独立。

不得混合计算。

------

# 14. 历史数据回补规则

## 14.1 回补起点

历史回补开始日期：

```text
2026-01-01
```

------

## 14.2 回补终点

系统部署当天。

------

## 14.3 回补逻辑

与日常采集使用：

```text
同一套解析逻辑
```

不得维护两套代码。

------

```text

```

------

# 16. 幂等性规则

## 16.1 重复采集

重复执行：

```text
同一个wr_id
```

不得产生：

```text
重复比赛

重复奖金

重复选手

重复队伍
```

------

## 16.2 保证方式

依赖：

```text
UNIQUE(wr_id)
```

以及业务校验实现。

------

# 17. 统计口径冻结

所有统计结果必须基于：

```text
首次采集数据
```

生成。

------

禁止：

```text
后续重新计算历史结果

后续覆盖历史数据
```

------
